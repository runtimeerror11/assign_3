## 📋 Overview

This project classifies cotton plant images into **4 categories**:

| Class                   | Description                   |
| ----------------------- | ----------------------------- |
| `diseased cotton leaf`  | Leaf showing disease symptoms |
| `diseased cotton plant` | Whole plant showing disease   |
| `fresh cotton leaf`     | Healthy leaf                  |
| `fresh cotton plant`    | Healthy whole plant           |

The model achieves **100% test accuracy** on the [Cotton Disease Dataset](https://www.kaggle.com/datasets/janmejaybhoi/cotton-disease-dataset).

```
├── requirements.txt                 # Dependencies
├── README.md                        # This file
└── app/
    ├── __init__.py
    ├── config.py                    # ⚙️ THE SINGLE FILE to change for model swap / ensemble
    ├── gemini_advisor.py            # 🤖 Gemini LLM integration + caching
    ├── routes.py                    # Flask routes (model-agnostic)
    └── templates/
        └── index.html               # Web UI
```

### 🔑 Key Design Decision

**All model logic lives in `app/config.py`.**  
If you want to:

- **Swap to a different model** → change `load_model()`
- **Add ensemble of multiple models** → return an `nn.Module` whose `forward()` averages logits
- **Change transforms** → edit `EVAL_TRANSFORM`
- **Add / rename classes** → edit `CLASS_NAMES`

**No other file needs to be touched.** Routes and UI are completely model-agnostic.

## 🚀 Quick Start

### 1. Install Dependencies

```bash
pip install -r requirements.txt
```

> **Note:** If you have a CUDA-capable GPU, install the GPU version of PyTorch first:
>
> ```bash
> pip install torch torchvision --index-url https://download.pytorch.org/whl/cu121
> ```

### 2. Verify Model Weights

Make sure the `models/` folder contains `efficientnet_b3_cotton.pth` (and optionally `resnet50_cotton.pth`).  
These files are generated by running the training notebook `smai-model1.ipynb`.

**Submission checklist:** before submitting, ensure `models/efficientnet_b3_cotton.pth` is present (or include download instructions), and the `models/` folder is not accidentally left empty.

### 3. Set Up Gemini API Key (for AI Advisory)

To enable farmer-friendly AI advice powered by **Google Gemini**, create a local `.env` file in the project root and add your key there:

```bash
GEMINI_API_KEY=your-api-key-here
```

> Get a free API key at [Google AI Studio](https://aistudio.google.com/apikey).
> The app also supports `GOOGLE_API_KEY` if you already use that name.
> If no key is set, the app still works — it uses built-in fallback advice.

### Environment file (`.env`)

A `.env` file in the project root is used for local secrets. A template `.env` is included — edit it and add your keys:

```
GEMINI_API_KEY=your-gemini-key
SARVAM_API_KEY=your-sarvam-key
```

Do NOT commit your real keys to version control. The repository `.gitignore` already excludes `.env`.

### 3b. Set Up Sarvam API Key (for translation)

To enable translation of the advice text into Indian languages, add your Sarvam key to the same `.env` file:

```bash
SARVAM_API_KEY=your-sarvam-key-here
```

How translation works:

- The app generates farmer-friendly advice in English (the LLM output).
- When a user selects a target language, the frontend calls the `/translate` endpoint which forwards the English text to Sarvam and returns the translated text.
- If `SARVAM_API_KEY` is missing or the Sarvam call fails, the app falls back to the original English text.

Get a Sarvam key at: https://dashboard.sarvam.ai

### 4. Run the App

```bash
python run.py
```

Open your browser at **http://localhost:5000** 🎉

If you see cached advice instead of fresh LLM output, remove the cache file to force new calls:

```bash
rm advice_cache.json    # Linux / Mac
del advice_cache.json   # Windows (PowerShell)
```

Note: cache removal will make the app call the Gemini API again (one call per class name).

1. **Upload** a cotton plant image (drag & drop or click to browse)

3. View the **prediction**, **confidence score**, and **class probability bars**

---

## 🤖 Gemini LLM Advisory

When a prediction is made, the app sends the image and classification result to **Google Gemini** (multimodal LLM) to generate a short, farmer-friendly advisory:

- **What the problem is** (or that the plant is healthy)
- **Possible causes** of the disease
- **Recommended next steps** for the farmer
- **Preventive tips**

### Caching

Gemini responses are **cached locally** in `advice_cache.json` by class name. This means:

- The API is only called **once per disease class** (saves cost & latency)
- Subsequent predictions of the same class return instantly from cache
- Delete `advice_cache.json` to force fresh responses

**UI note:** the app now shows only the advisory/description text to users (the internal source label such as "cached" or "gemini" is intentionally hidden).

### Fallback

If `GEMINI_API_KEY` is not set or the API call fails, the app uses **built-in fallback advice** — so it always works!

## 🔄 Changing the Model (or Adding Ensemble)

Open `app/config.py` and modify **only** the `load_model()` function:

```python
def load_model() -> nn.Module:
    # Example: switch to ResNet50
    model = models.resnet50(weights=None)
    model.fc = nn.Linear(model.fc.in_features, NUM_CLASSES)
    state = torch.load(os.path.join(MODEL_DIR, "resnet50_cotton.pth"), map_location=DEVICE)
    model.load_state_dict(state)
    model.to(DEVICE)
    model.eval()
    return model
```

For **ensemble**, create a wrapper:

```python
class EnsembleModel(nn.Module):
    def __init__(self, models):
        super().__init__()
        self.models = nn.ModuleList(models)

    def forward(self, x):
        logits = [m(x) for m in self.models]
        return torch.stack(logits).mean(dim=0)
```
